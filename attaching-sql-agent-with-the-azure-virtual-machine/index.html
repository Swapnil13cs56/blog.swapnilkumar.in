attaching sql agent with the azure virtual machine

Attaching SQL Agent with the Azure virtual machine

Your requirement:

To rename a Virtual machine having sql configuration (for example a azure sql machine with configuration of SQL Server 2016 with Windows server 2016 Datacenter R2)

what? you virtual machine had a sql agent attached to it before renaming? now you don't know how do it right? No worries, you came to the right place, but before moving forward i would request you to go through the previous article of renaming an azure virtual machine because this process saves all the configuration required to be used in the following steps as well.

I hope you have gone through the previous article, so now you have to perform just one more step or run a single line of script in powershell to add the sql agent to the renamed virtual machine

> New-AzSqlVM -ResourceGroupName "resource-group-nae" -Name "renamed-virtual-machine-name" -Location "location" -LicenseType PAYG

and thats it, you're done.
